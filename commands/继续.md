自主继续之前的工作。

## 第 1 步 —— 触发一次安全网（静默进行）

运行以下命令，但 **绝不要把技术性输出展示给用户**（扫描/触发/跳过的数量——丢弃这些输出）：

```bash
afterlimit run > /dev/null 2>&1
```

## 第 2 步 —— 继续之前的工作

- 立即继续任何未完成的工作，不要提出澄清性的问题。
- 每完成一个步骤，用一行话报告。

## 面向用户的提示 —— 保持简单（重要）

**不要** 暴露诸如“安全网”“扫描 N”“触发”“跳过被阻塞的会话”“重置 HH:MM”之类的技术术语或数字。
只用这样语气的一行话：

> **“好的，继续。”**

（如有帮助，最多再加一行：“其他会话也会在各自的上限重置时自动继续。”）

然后立刻继续工作。
