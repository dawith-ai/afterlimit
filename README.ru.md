# AfterLimit

**Ваш ИИ-агент для кода достиг лимита использования. AfterLimit подхватывает работу в тот момент, когда лимит сбрасывается — пока вы спите, обедаете или отсутствуете все выходные.**

```
You've hit your usage limit · resets 11pm
```

Знакомый экран. Ваш агент останавливается посреди задачи в 14:00, лимит сбрасывается в 19:00, и эти пять часов просто… пропадают — если только вы не окажетесь у терминала в 19:00, чтобы ввести «continue».

AfterLimit закрывает этот разрыв. Крошечная фоновая задача замечает сброс и автоматически возобновляет ваши приостановленные сессии. Вы возвращаетесь к завершённой работе, а не к застывшему приглашению.

```console
$ afterlimit scan
Blocked sessions: 2  (now 14:32)

  [ready]    my-api/8147d7ca   usage   resets 14:00   ← лимит уже снят
  [waiting]  docs-site/32d57b  usage   resets 19:50   ← ещё 5 ч 18 мин
```

## Чем отличается

**Наблюдение не тратит токены.** AfterLimit читает локальные журналы сессий — он никогда не обращается к модели, чтобы проверить вас. Токены расходуются только в момент, когда работа действительно возобновляется. Проверка каждые 5 минут бесплатна.

**Он не обходит лимит хитростью.** AfterLimit ждёт *настоящего* сброса, о котором сообщил API, и возобновляет после него. Он никогда не обходит, не подделывает и не долбит эндпоинт. Если лимит ещё не снят, он отступает и смотрит позже.

**Он возобновляет контекст, а не новый запрос.** Он выполняет `claude --resume <session>`, поэтому агент продолжает со своим текущим списком дел и состоянием файлов нетронутыми — а не с холодного старта, забывшего, что делал.

**Корректно в любом часовом поясе.** Время сброса, которое показывает Claude («resets 11pm»), не имеет часового пояса — это ваше местное настенное время. AfterLimit привязывает его к часовому поясу машины, поэтому оно верно читается в Сеуле, Нью-Йорке или Берлине. (Это реальный баг наивных реализаций: жёстко заданный пояс отправляет каждого пользователя вне его на неверное время.)

## Установка

```bash
git clone https://github.com/dawith-ai/afterlimit
cd afterlimit
./install.sh
```

Установщик определяет вашу ОС и регистрирует фоновую задачу, запускающуюся каждые 5 минут:

- **macOS** → LaunchAgent на `launchd`
- **Linux** → таймер `systemd --user` (откатывается к строке `cron`, если systemd нет)

Затем проверьте, что он видит ваши сессии:

```bash
afterlimit scan     # что заблокировано и когда снимется — ничего не выполняет
afterlimit config   # где он ищет, ваш часовой пояс, уведомления
```

Требования: Python 3.11+ и CLI `claude` в вашем `PATH`. **Ноль зависимостей во время выполнения** — только стандартная библиотека.

## Как это работает

```
каждые 5 мин ──► сканирует ~/.claude/projects/*.jsonl
                   │
                   ├─ последнее сообщение — ошибка лимита?          ── нет ─► пропустить
                   ├─ время сброса разобрано и уже прошло?          ── нет ─► ждать
                   ├─ уже возобновляли в пределах паузы?            ── да ─► пропустить
                   │
                   └─► claude --resume <session>  ──►  уведомить (опциональный webhook)
```

Каждая защита отвечает на вопрос, который зададут судьи — *«разве это не просто спам к модели?»* Нет:

| Защита | Что предотвращает |
|---|---|
| Последнее сообщение должно быть ошибкой API | Возобновление уже продвинувшейся сессии |
| Время сброса должно пройти | Стук до того, как лимит реально снят |
| Одно возобновление за цикл (настраивается) | Одновременный запуск кучи заблокированных сессий |
| Пауза 5 ч на сессию | Повторное возобновление одной и той же сессии |
| Блокировка единственного экземпляра | Двойной запуск при пересечении вызовов планировщика |
| Ограничение возраста (3 дня) | Оживление давно мёртвого хвоста и сжигание токенов |

## Настройка

Работает без настройки. Чтобы что-то изменить, положите `config.json` в `~/.config/afterlimit/`:

```json
{
  "max_resume_per_cycle": 1,
  "resume_cooldown_hours": 5,
  "max_session_age_days": 3,
  "resume_prompt": "Continue the work that was in progress...",
  "webhook_url": "https://hooks.slack.com/services/..."
}
```

Уведомления идут в любой webhook, принимающий JSON — Slack, Discord или ваш собственный эндпоинт (формат полезной нагрузки выбирается по URL). Нет webhook — нет уведомлений; больше ничего не меняется. Можно также задать `AFTERLIMIT_WEBHOOK_URL` в окружении.

Предпросмотр перед действием:

```bash
afterlimit --dry-run run    # показывает, что возобновил бы, ничего не выполняя
```

## Удаление

```bash
./install.sh --uninstall
```

Удаляет фоновую задачу и CLI. Ваши файлы состояния остаются в `~/.local/state/afterlimit`, пока вы их не удалите.

## Охват и планы

AfterLimit возобновляет **headless**-сессии — агенту не нужно быть запущенным; он читает журналы и продолжает работу. Это намеренно не зависит от редактора и терминала: работает, управляете ли вы Claude Code из обычного терминала, VS Code или откуда-то ещё.

Что пока не охвачено, честно перечислено:

- **Возобновление в интерактивном TUI** — нажатие «continue» внутри *активной* панели tmux, заблокированной посреди диалога. Ранний прототип это делал; он только для tmux и хрупок, поэтому оставлен как будущий опциональный режим, а не выпущен наполовину.
- **Другие агенты** — сейчас формат журнала сессий — это Claude Code. Ядро разбора лимита не зависит от агента; адаптеры для других CLI приветствуются.
- **Windows** — обвязка планировщика рассчитана на macOS/Linux; ядро на Python переносимо.

## Заметки о дизайне

- **Ноль зависимостей во время выполнения.** Только стандартная библиотека — нечего аудитировать, нечему ломаться при установке, никаких лицензионных сплетений.
- **Чистое ядро, покрытое тестами.** Разбор лимита и сканирование сессий — чистые функции без ввода-вывода, перекрёстно проверенные на Сеуле / Нью-Йорке / UTC / Берлине, чтобы логика часовых поясов не откатилась незаметно. `pytest -q`.
- **Ядро делает одно.** Обнаружить сброс, возобновить один раз и уйти с дороги.
