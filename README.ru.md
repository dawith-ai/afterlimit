# Dawith Claude terminal auto (Mac)

> **Больше никогда не сидите с Claude Code из-за лимита использования.** Когда Claude Code достигает лимита использования, этот фоновый инструмент для macOS автоматически выбирает **«Stop and wait for limit to reset»** — и ваша работа возобновляется сама собой в тот момент, когда лимит сбрасывается, даже если вас нет рядом.

```
What do you want to do?
❯ 1. Stop and wait for limit to reset
  2. Upgrade your plan
```

Больше не нужно сидеть у терминала, чтобы нажать **«1»** в этом меню. Просто уйдите — работа возобновится при сбросе лимита, а затем вам придёт уведомление в **Discord / Telegram / Slack**, чтобы вы знали, что она продолжилась.

**Весь процесс**: передайте git-ссылку Claude Code → `install.sh` → устанавливается команда `/continue` → вы запускаете `/continue` и уходите → при сбросе токенов ваша работа возобновляется автоматически → вы получаете уведомление в мессенджере.

## Главная идея: наблюдение не стоит ни одного токена

Наблюдатель — это **локальный Python, который лишь сканирует ваш экран и файлы** — он никогда не обращается к Claude, поэтому тратит ноль токенов (= ноль затрат). Токены расходуются только в тот момент, когда работа **фактически возобновляется**, и лишь один раз за сессию при сбросе. «Разве запуск каждую минуту не дорого?» → опрос бесплатен; платно только возобновление.

## Две страховочные сети

| Name | Target | Action | Interval |
|---|---|---|---|
| **tmux-resume** (core) | активные сессии терминала внутри tmux | обнаруживает меню лимита → автоматически выбирает вариант 1 «Stop and wait for limit to reset» (`1` → `Enter`) | 60s |
| **resume-safety** (backup) | оставленные сессии, от которых вы отошли | сканирует журналы диалогов (jsonl) → `claude --resume` в фоне при сбросе | 300s |

Они не пересекаются: `resume-safety` **уступает** проектам, у которых есть активная сессия (чтобы не конкурировать за одну и ту же квоту аккаунта), а тем терминалом занимается `tmux-resume`.

## Режим: как далеко идёт автопродолжение

Вы выбираете через `resume_mode` в `~/.config/claude-terminal-auto/notify.json`:

| Mode | Behavior | Tokens |
|---|---|---|
| **`token_only`** (по умолчанию, рекомендуется) | обрабатывает только меню лимита использования. Останавливается и спрашивает, когда задача завершена. | экономно |
| **`keep_going`** | всё вышеперечисленное, плюс **автоматически подталкивает простаивающие сессии продолжать** после их завершения → никогда не останавливается ночью. | продолжает тратить |

```jsonc
{ "resume_mode": "token_only" }   // or "keep_going"
```

Защитные меры `keep_going`: никогда не трогает сессию, которая генерирует ответ или имеет черновик в поле ввода; кулдаун 15 минут на каждую панель. (Она продолжает работать автономно и тратит токены, поэтому включайте её только когда вам это действительно нужно.)

## Требования

- **macOS** (launchd)
- **tmux** — сессии должны выполняться внутри tmux для внедрения нажатий клавиш (macOS блокирует внедрение клавиш вне tmux)
- **Python 3** — только стандартная библиотека, ничего устанавливать не нужно

## Установка

```bash
git clone https://github.com/dawith-ai/Dawith-Claude-terminal-auto-Mac.git
cd Dawith-Claude-terminal-auto-Mac
./install.sh
```

`install.sh` переписывает пути в plist на эту папку, устанавливает их в `~/Library/LaunchAgents` и регистрирует в launchd (сохраняется после перезагрузки). Если вы используете Claude Code, **slash-команда `/continue`** (и её переводы) также устанавливается в `~/.claude/commands/`. Проверьте статус:

```bash
launchctl list | grep claude-terminal-auto
```

## Удаление

```bash
./uninstall.sh
```

## Slash-команда `/continue` (Claude Code)

Отдельно от фонового наблюдателя, ввод **`/continue`** в Claude Code единожды запускает страховочную сеть и возобновляет прерванную работу.

- **Фоновое авто** (launchd) = автоматически при сбросе, вводить ничего не нужно
- **`/continue`** (slash) = ручной запуск для тех случаев, когда вы хотите начать немедленно, а не ждать

Локализованные названия команд установлены для каждого языка, так что вы можете использовать её на своём:

| Language | Command | Language | Command |
|---|---|---|---|
| English | `/continue` | Español | `/continuar` |
| 한국어 | `/지속` | Français | `/continuer` |
| 中文 | `/继续` | Deutsch | `/weiter` |
| 日本語 | `/続行` | Português | `/prosseguir` |
| Русский | `/продолжить` | | |

## Уведомления в мессенджерах (Discord / Telegram / Slack / любой вебхук)

Получайте уведомление, когда работа возобновляется автоматически. `install.sh` создаёт шаблон в `~/.config/claude-terminal-auto/notify.json`; **заполните только те каналы, которые вам нужны** (остальные оставьте пустыми, чтобы они были отключены).

```jsonc
{
  "resume_mode": "token_only",
  "discord_webhook": "https://discord.com/api/webhooks/...",   // Discord Incoming Webhook
  "telegram_token": "123456:ABC...",                            // Telegram bot token
  "telegram_chat_id": "12345678",                               // Telegram chat id
  "slack_webhook": "https://hooks.slack.com/services/...",      // Slack Incoming Webhook
  "generic_webhooks": [                                         // any other messenger, no code needed
    { "name": "mattermost", "url": "https://.../hooks/xxx", "field": "text" }
  ]
}
```

- **Добавить больше мессенджеров, двумя способами**:
  1. **Без кода** — добавьте `{url, field, name}` в `generic_webhooks` (работает с Mattermost, Google Chat, Slack-совместимыми и большинством сервисов, принимающих JSON POST)
  2. **Отдельная функция** — добавьте функцию `_send_*` в `scripts/notify.py` и одну строку в `_SENDERS` (для особых форматов)
- Также настраивается через переменные окружения: `CLAUDE_AUTO_DISCORD_WEBHOOK` / `CLAUDE_AUTO_TELEGRAM_TOKEN` / `CLAUDE_AUTO_TELEGRAM_CHAT_ID` / `CLAUDE_AUTO_SLACK_WEBHOOK`
- Проверка: `python3 scripts/notify.py "test"` → отправляет в настроенные каналы
- ⚠️ `notify.json` содержит токены/вебхуки, поэтому он исключён из git (репозиторий поставляется только с пустым `notify.example.json`).

## Как это работает

- **tmux-resume** (`scripts/tmux_resume_watcher.py`): считывает каждую панель с помощью `tmux capture-pane`; когда меню лимита (`What do you want to do? / 1. Stop and wait for limit to reset / 2. Upgrade your plan`) **активно** внизу, использует `tmux send-keys`, чтобы нажать **`1` → `Enter`**, подтверждая вариант 1 → Claude возобновляет вашу работу при сбросе. (⚠️ Он никогда не отправляет `Esc`, который отменил бы меню.) **Защита от ложных срабатываний**: срабатывает только когда присутствуют все три фразы меню и при этом нет обычной строки ввода (`bypass permissions`) / состояния генерации (`esc to interrupt`); кулдаун 5 минут на каждую панель.
- **resume-safety** (`scripts/resume_blocked_sessions.py`): сканирует журналы диалогов в `~/.claude/projects`, находит сессии, заблокированные лимитом, и возобновляет их новым процессом `claude --resume` при сбросе. У неё есть **меры экономии** (одно возобновление за 5-часовое окно сессии, порог уступки по использованию сессии). Для headless-запусков она считывает OAuth-токен Claude из связки ключей macOS во время выполнения — он никогда не хранится в исходном коде.

## Примечания

- Этот инструмент предполагает **автономное выполнение** («продолжай без вопросов»), когда он подтверждает меню. Учитывайте это при работе с чувствительными задачами.
- Если вы уже запускаете те же скрипты под другой меткой (например, `com.openclaw.*`), **не устанавливайте их дважды** (это позволяет избежать двойного срабатывания).
